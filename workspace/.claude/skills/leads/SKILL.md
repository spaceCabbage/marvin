# Lead Qualification Skill

Expert B2B lead research, qualification, and discovery for tech consulting services. Build detailed company dossiers with specific inefficiencies identified, the right decision makers to contact, and tailored pitch strategies.

## Trigger

When user asks about:
- Lead qualification, lead research, prospect research
- Company dossier, company research, company intel

- "Find leads", "Discover prospects", "Find companies to target"
- "Score these leads", "Qualify these companies"

## Ideal Customer Profile (ICP)

### Target Company Profile

| Criteria      | Target                                            |
|---------------|---------------------------------------------------|
| **Size**      | 15-100 employees                                  |
| **Geography** | USA only                                          |
| **Bonus**     | Tristate area (NY/NJ/CT), Jewish-owned businesses |

### Target Industries

Focus: **Office-based operations that overuse spreadsheets, copy-paste workflows, and manual data entry**

#### High Priority (+15 points)
- **Logistics / 3PL** - Dispatch, routing, tracking all spreadsheet-heavy
- **Property Management** - Tenant tracking, maintenance, rent collection
- **E-commerce Operations** - Multi-channel inventory, order management
- **Wholesale Distribution** - Orders, inventory, invoicing
- **Insurance Agencies** - Quote management, claims tracking, renewals
- **Mortgage / Lending** - Loan tracking, document management
- **Freight Brokers** - Load matching, carrier management
- **Staffing / Recruiting Agencies** - Candidate tracking, placement management

#### Medium Priority (+10 points)
- **Medical Billing / RCM** - Claims, denials, AR management (not clinical)
- **Accounting Firms** - Client management, document workflow
- **Legal Admin** - Case management, billing, document handling
- **Construction Admin** - Project tracking, subcontractor management, billing
- **HOA Management** - Resident management, violation tracking, dues
- **Commercial Real Estate** - Lease management, tenant relations
- **Auto Dealerships** - Inventory, financing paperwork, service scheduling
- **Equipment Rental** - Inventory, reservations, maintenance tracking

#### Lower Priority (+5 points)
- **Event Planning** - Vendor management, scheduling, budgets
- **Travel Agencies** - Booking management, itineraries
- **Nonprofit Admin** - Donor management, grant tracking
- **Education Admin** - Enrollment, scheduling, billing (not teaching)
- **Funeral Homes** - Service coordination, pre-planning, AR

**SKIP These (not office/computer based):**
- Manufacturing floor work
- Agriculture field work
- Construction trades (hands-on)
- Healthcare clinical (doctors, nurses)

**USUALLY SKIP (robust off-the-shelf software exists):**
- **Basic sales teams** - Salesforce/HubSpot handles 90% of CRM needs
- **Simple e-commerce** - Shopify is "good enough" for most
- **Standard accounting** - QuickBooks/Xero covers basic needs
- **Basic HR/payroll** - Gusto, ADP, Rippling are solid
- **Marketing teams** - HubSpot, Mailchimp, etc. are mature
- **Simple project mgmt** - Asana, Monday, Notion work fine
- **Standard inventory** - Many good SaaS options exist

**EXCEPTION: These industries ARE worth pursuing IF:**
- They're using 3+ disconnected tools (Shopify + separate inventory + separate shipping + QuickBooks = integration opportunity)
- They have unique workflows the off-the-shelf can't handle
- They're doing heavy copy-paste between systems
- They've outgrown their tools but are stuck with manual workarounds
- They mention "we use Zapier for everything" (band-aid city)
- Industry-specific needs the generic tools don't address

**Key Question**: "Is their pain from LACK of software, or from software that doesn't TALK to each other?"
- Lack of software = probably off-the-shelf exists, skip
- Software doesn't talk = integration opportunity, pursue

- 0-39: **COLD** - Not a fit

## Phase 0: Engagement Setup (ALWAYS DO FIRST)

When the skill is triggered (e.g., `/leads` or "research company X"):

### Step 1: Gather Initial Info (Use AskUserQuestion)

If user didn't provide company details, **ASK**:

> "Let's qualify a lead. I need some info:
>
> **Company name or domain?** (e.g., 'Acme Logistics' or 'acmelogistics.com')
>
> **What do you know about them?** (industry, how you found them, etc.)
>
> **What's the goal?** (quick score, full dossier, or both?)"

### Step 2: Set Up Workspace

```bash
# Slugify company name
COMPANY_SLUG=$(echo "[company]" | tr '[:upper:]' '[:lower:]' | tr ' ' '-' | tr -cd '[:alnum:]-')
DATE=$(date +%Y-%m-%d)
WORKSPACE="$HOME/engagements/${COMPANY_SLUG}/lead_qual_${DATE}"

# Create directory structure
mkdir -p "${WORKSPACE}"/{report,raw/{leadership,inefficiencies,solution_fit,intel,website_scrape},evidence/screenshots}

# Set current engagement for status line
echo "${COMPANY_SLUG}" > ~/.current-engagement
```

### Step 3: Domain Discovery & Website Scrape

If only company name provided, find domain:
```bash
# Web search: "[company name] official website"
# Extract domain from results
```

**Immediately scrape the company website (use Playwright MCP or websearch):**
- Homepage content
- About Us page
- Team/Leadership page
- Careers/Jobs page
- Contact page
- Any technology/integrations mentioned

Save all scraped content to `${WORKSPACE}/raw/website_scrape/`

### Step 4: WHOIS & Domain Intel

```bash
# Get domain registration info
whois [domain] > "${WORKSPACE}/raw/intel/whois.txt"

# Check: registrant info often reveals owner name!
# Also: domain age, registrar, contact email adnb other domain based lookup osint tools
```

## Email Verification (CRITICAL)

**A lead without a verified email for the attack person is not a quality lead.**

### Rules

1. **Only use VERIFIED emails** - emails we can confirm actually exist via OSINT tools
2. **NEVER use guessed patterns** like `firstname@domain.com`, `first.last@domain.com`, `f.lastname@domain.com`
3. If we can't verify the email, mark the contact as **"email unverified"**
4. Unverified emails should NOT be used for outreach - they hurt deliverability and look unprofessional

### Verification Methods

**Use these tools to verify emails exist:**

```bash
# Check if email is registered on various services (proves it exists)
holehe target@company.com

# Check breach databases (if email appears in breaches, it exists)
h8mail -t target@company.com

# Find actual email patterns from public sources
theHarvester -d company.com -b all

# Check social platforms
socialscan target@company.com
```

### Confidence Levels

| Level               | Criteria                                                                  | Action             |
|---------------------|---------------------------------------------------------------------------|--------------------|
| **Verified**        | Found in breach DB, registered on services, or scraped from public source | ✅ Use for outreach |
| **High Confidence** | Pattern matches multiple verified emails at same domain                   | ✅ Use with caution |
| **Guessed**         | Pattern-based guess (firstname@domain.com) with no verification           | ❌ Do NOT use       |

### In the Dossier

Always indicate email verification status:

```
**Attack Person:** John Smith, COO
- LinkedIn: linkedin.com/in/johnsmith
- Email: john.smith@acme.com ✅ (verified via holehe - registered on LinkedIn, Slack)
```

Or if unverified:

```
**Attack Person:** John Smith, COO
- LinkedIn: linkedin.com/in/johnsmith
- Email: ❌ Not verified (pattern guess only - do not use)
- **Recommended:** Connect on LinkedIn first
```

### Why This Matters

- Guessed emails bounce → hurts sender reputation
- Wrong person receives email → looks unprofessional
- Cold email to unverified address → likely spam filtered
- LinkedIn connection request is better than guessed email

## Best Practices

### Research Discipline

1. **Evidence over assumptions** - Every inefficiency claim needs a source
2. **Specific over generic** - "They have 3 data entry clerks" not "they might have manual processes"
3. **Score honestly** - Don't inflate scores for marginal leads
4. **Update as you learn** - Store new info in SQLite for future reference
5. include links to sources by each data mention or use foototes in the output

### User Involvement (ASK EAGERLY)

- **Before deep dives**: "Quick score is 72. Want full dossier?"
- **At decision points**: "Found 3 potential contacts. Which to research first?"
- **Discovery criteria**: "What industries/regions should I focus on?"
- **Batch size**: "Found 50 companies. Qualify all or top 20?"

### Efficiency

1. **Quick score first** - Don't waste time on obvious non-fits
2. **Parallel subagents** - *Always* spawn research in parallel
3. **Store everything** - SQLite lets us query past research
4. **Reuse intel** - Check if we've researched this company before
